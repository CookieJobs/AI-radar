# Radar — 产品方案 v0.2

> 版本：v0.2（2026-06-18 重写，从 v0.1 砍订阅改纯 B2A）
> 状态：MVP 已跑通（49 条种子事件 + REST API + MCP server），进入冷启动期
> 品牌：**Radar**（全网统一，无前缀）

---

## 〇、版本变化（vs v0.1）

v0.2 的核心转向：**从"B2A 订阅"切到"纯 B2A 按调用"**。

| 维度 | v0.1 | v0.2 |
|---|---|---|
| 产品形态 | 实时 API + **每日订阅周报** + 深度问答 API | **仅 API**（实时事件 + 深度问答两条 SKU） |
| 文章化分发 | 公众号 + 飞书 + 即刻 + X 同步 | **全部砍掉**，不投入人读版内容 |
| 付费模型 | ¥39 / ¥199 月付订阅 + ¥0.1-0.5/次 API | **冷启动期全部免费**，等微信/支付宝 agent 钱包上线再按调用付费 |
| 客户定义 | "AI PM、OPC 投资人订阅者" | "**agent 的主人**"——他为 agent 的能力付费，agent 是消费实体 |
| 品牌名 | AI Radar（候选） | **Radar**（已定） |

**核心判断**：周报是个人 IP 时代的产物。**Radar 卖的不是"内容给读者"，是"情报给 agent 调"**。人读版获客路径长、转化弱、对个人作者是负担——直接砍。

---

## 一、产品一句话

**Radar 是 B2A（Business to Agent）模式的 AI 行业情报 API。**

agent 调一次 `GET /v1/events`，拿到结构化的事件 + 解读 + 推演。agent 的主人按调用付费，agent 拿到"行业今天发生了什么 + 这意味着什么 + 接下来 30 天会怎样"。

---

## 二、为什么这件事现在成立

### 2.1 三个趋势的共振

| 趋势 | 时间窗口 | 我们的窗口期 |
|---|---|---|
| 微信 / 支付宝 agent 钱包上线 | 2026 H2 - 2027 H1 | **6-12 个月先发优势** |
| WorkBuddy 等桌面级 agent 普及 | 2026 - 2027 | 已开始 |
| 通用大模型实时信息能力不足 | 持续 | 长期 |

**关键判断**：2026 H2 是"agent 数据源"赛道的窗口期。**先入场者吃到红利，后入场者只能吃残羹。**

### 2.2 谁会"通过 agent"调用我们

冷启动期三类 agent 主人：

1. **AI 产品经理同行的个人 agent**
   - 他们的 agent 需要持续喂行业信息（他们不会自己每天刷 50 个信源）
   - 付费意愿高，时间稀缺
   - **他们买的是"判断"，不是"事实"**

2. **OPC 创业者的运营 / 调研 agent**
   - 信息不对称是最大焦虑
   - agent 调一次，比雇一个分析师便宜
   - **他们买的是"不要错过赛道变化"**

3. **早期投资人 / FA 的尽调 agent**
   - 决策密集，agent 帮人扫 + 解读
   - 单笔 ARPU 高，对价格不敏感
   - **他们买的是"深度 + 持续"**

## 三、产品形态

### 3.1 两条 SKU

#### SKU 1：实时事件 API（B2A 主打）

```
GET /v1/events?domain=llm&since=7d&min_impact=4

Response:
{
  "events": [
    {
      "id": "evt_2026_0617_anthropic_claude45",
      "timestamp": "2026-06-17T08:30:00Z",
      "domain": "llm",
      "category": "model_release",
      "entities": ["Anthropic", "Claude"],
      "headline": "Anthropic 发布 Claude 4.5，定位 agent 工作流",
      "summary_fact": "（200 字事实摘要）",
      "summary_insight": "（200 字专家解读，对 OPC 创业者意味着什么）",
      "summary_forecast": "（100 字未来 30 天推演）",
      "impact_level": 4,
      "confidence": 0.85,
      "source": [{"name": "...", "url": "..."}],
      "tags": ["model_release", "agent", "programming"]
    }
  ]
}
```

**冷启动定价：免费 / 限速**（每分钟 10 次 / 每 IP 1000 次/天）。等 agent 钱包上线后改按调用计费 ¥0.1-0.5/次。

#### SKU 2：深度问答 API（B2A 高溢价）

- agent 可以问："X 公司最新动态？给我深度分析"
- 后端用 RAG + 人工定期补料
- **冷启动定价：免费**（限速更严，每分钟 2 次）
- 后续定价：5 credits/次 = ¥0.5/次

#### 砍掉的 SKU

- ❌ **每日简报订阅**（v0.1 SKU 2）——不做人读版
- ❌ **付费墙**（v0.1 摘要公开 / 全文订阅）——全部免费，付费是 agent 钱包的事

## 四、内容生产 SOP（**只服务 API，不服务订阅**）

### 4.1 节奏

- **每周 5-8 条事件**（不是"周报"，是"事件流"）
- **生产周期**：日更（每天 1-2 条）
- **覆盖窗口**：过去 24-72 小时

### 4.2 SOP 详解

**每天早 8:00-10:00：AI 扫描 + 选题**
- AI 跑 50 个信源，输出 5-10 条候选
- 你花 30 分钟筛出 2-3 条进入深度候选

**每天晚 20:00-22:00：你写作 + 终审**
- 你亲自写 200 字事实摘要 + 200 字解读 + 100 字推演
- AI 不参与"我的判断"
- AI 只做格式校验、JSON 转换、字段补齐

**发布即同步**——agent 调 API 时直接拿到最新事件。**没有"周三发布"这种统一动作**。

**每日时间投入：1.5-2 小时**（比 v0.1 每周 8-12 小时低，因为是日更分散 + 砍掉文章化）

## 五、AI 与人的分工

### 5.1 AI 能做（让它做）

| 任务 | 工具 | 价值 |
|---|---|---|
| 原始信息抓取 | RSS / API / scraper | 覆盖 50+ 信源不疲劳 |
| 事实摘要草稿 | LLM | 200 字摘要质量稳定 |
| 多源交叉验证 | LLM | 减少错数据风险 |
| 简单标签分类 | LLM + 规则 | 节省 30 分钟/篇 |
| API JSON 转换 | LLM | 一次生产双 API SKU |

### 5.2 你必须做（不让 AI 做）

| 任务 | 为什么必须人做 |
|---|---|
| 选题筛选（这事儿重要吗？） | AI 没价值观，不会选 |
| 解读（这意味着什么？） | AI 给的是"标准答案"，你要给"个人判断" |
| 推演（接下来 30 天怎么演化？） | AI 没预测能力，没人 + 跨域视角不行 |
| 质量终审（这条值不值得 agent 付费？） | 这是你对用户的承诺 |

## 六、商业模式（纯 B2A）

### 6.1 三个阶段

| 阶段 | 时间 | 付费策略 |
|---|---|---|
| 冷启动 | T0 - T+90d | **完全免费 / 限速**，跑真实调用数据 |
| 钱包上线 | T+90d - T+180d | 接微信 / 支付宝 agent 钱包，按调用 ¥0.1-0.5/次 |
| 规模化 | T+180d+ | 多档定价 + 企业 SLA + WorkBuddy Skill 上架 |

### 6.2 关键原则

- **不做人读版**——不烧获客成本在"读者"上，烧在"agent 接入"上
- **agent 友好**——¥0.1/次级别，否则 agent 不会调
- **先跑调用再定价**——30 天免费期后看实际调用分布，再定阈值
- **不绑年付 / 不绑月付**——按调用，让 agent 自己判断值不值

### 6.3 v0.1 三档定价 → v0.2 全部作废

| v0.1 档位 | v0.2 处置 |
|---|---|
| 免费（200 字摘要） | **取消**——冷启动全部免费，但**没有付费墙** |
| 个人订阅 ¥39/月 | **作废** |
| 团队订阅 ¥199/月 | **作废** |
| Agent API ¥0.1-0.5/次 | **保留**，延后到钱包上线再启用 |
| 企业 API ¥999/月起 | **保留**，延后到规模化阶段 |

## 七、商业道德红线

### 7.1 不能碰的

- ❌ 高途未公开的用户数据、教学内容、产品数据
- ❌ 客户名单、未公开产品规划、未公开财务
- ❌ 公司内部培训资料、保密协议覆盖的素材
- ❌ 任何违反 NDA / 保密协议的素材
- ❌ 抄袭未授权的研报、报告、文章

### 7.2 可以做的

- ✅ 公开信源抓取 + 二次解读
- ✅ 个人方法论（PRD 模板、访谈脚本、评审清单）
- ✅ 自购素材二次加工（不传播盗版）
- ✅ 公开数据集整理 + 标签化

## 八、风险与对策

| 风险 | 概率 | 影响 | 对策 |
|---|---|---|---|
| 内容质量不稳 | 高 | 高 | 强制质量 checklist + 试读机制 |
| 大模型自己变得实时 | 中 | 高 | 我们卖判断不卖事实，护城河更深 |
| 信源被限流/付费墙 | 中 | 中 | 多源 + 备份 + 自建 RSS |
| WorkBuddy Skill 上架审核不通过 | 中 | 中 | 同时上架 MCP / 自建 API |
| 微信/支付宝 agent 钱包推迟 | 低 | 中 | **冷启动期完全免费，不依赖付费时间表** |
| 化名人设翻车 | 低 | 高 | 不实名、不爆照、不引战；About 只写"AI 产品经理"，**不暴露公司名** |
| 免费期被 agent 拉爆服务器 | 中 | 中 | 严格限速 + 监控异常调用方 |
