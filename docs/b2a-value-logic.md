# B2A 价值逻辑 + 4 层护城河

> 这是 AI Radar 产品设计的最底层逻辑。
> 如果这一层的逻辑没想清楚，所有上层设计都是空中楼阁。

---

## 一、为什么 B2A 卖情报能成立

### 1.1 客户变了

| 维度 | 传统卖研报 | B2A 卖情报 |
|---|---|---|
| 买方 | 人类分析师 | Agent |
| 使用方式 | 看完写报告 | 直接消费 |
| 时效要求 | 月/季 | 小时/天 |
| 决策链路 | 人读 → 人判断 → 人决策 | agent 读 → 喂给主人 → 人决策 |
| 核心价值 | "我替你读" | "让我的 agent 变专业" |

**关键转变：** 客户从"人"变成了"agent"——这意味着：
- 交付物必须是**机器可读**
- 时效必须**实时**
- **单次调用价值要被定价**（不是按月订阅）

### 1.2 单次 API 调用的价值公式

> **单次 API 调用的价值 = 它替 agent 主人省下的决策时间 × 决策失误的代价**

**举例 1（典型场景）：**
- 场景：OPC 主理人的 agent 问"我应该升级到 Claude 4.5 吗？"
- 你的 API 给出答案 + 解读 + 推演
- 价值：
  - 省 2 小时研究（按 ¥100/h = ¥200）
  - 避免错误采购（错买可能浪费 ¥500/月）
  - **总价值：¥700/次**
- **你的定价：¥0.5/次**（用户月节省的 1/1400）
- 用户感受：**完全无痛付费**

**举例 2（高价值场景）：**
- 场景：投资人的 agent 问"X 公司最近融资进展？"
- 你的 API 给出答案 + 影响等级 + 推演
- 价值：
  - 错失投资机会损失 ¥10000+
  - 错误投资损失 ¥50000+
  - **总价值：¥100000+/次**
- **你的定价：¥5/次**（用户月节省的 1/20000）
- 用户感受：**依然超值**

**核心洞察：** 单次 API 价值远高于你收的价格——**只要你的数据准 + 判断对**，用户会持续付费。

## 二、4 层护城河

**核心论点：你的产品不是 L1 数据，是 L3 + L4 解读 + 推演。**

### 2.1 L1：原始事实

- 例子："Anthropic 发布 Claude 4.5"
- 信息源：Anthropic 官方博客
- 价值：**几乎为 0**
- 谁都能爬，谁都能转述
- 替代品：Google News、Twitter、RSS

### 2.2 L2：标签 + 分类

- 例子：
  ```
  事件：Claude 4.5 发布
  标签：model_release / agent / programming
  重要性：4/5
  可信度：0.85
  ```
- 价值：**¥0.01/次**
- AI 能做，但需要"标准制定者"
- 替代品：自建分类体系需要时间

### 2.3 L3：解读（这意味着什么）

- 例子：
  > 对 OPC 创业者意味着：编程 agent 成本可控 +30%，短期内不必切换主模型。
- 价值：**¥0.1-0.5/次**
- AI 能给"标准答案"，但**真人能给出"我的判断"**
- 差异化：你个人的解读框架 + 跨域视角

### 2.4 L4：推演（接下来 30 天怎么演化）

- 例子：
  > 未来 30 天：OpenAI / Google 大概率跟进 agent 化升级，Cursor / Devin 类产品可能调价。
- 价值：**¥1-5/次**
- AI 做不了（没预测能力）
- 必须靠人 + 跨域视野 + 历史模式识别

### 2.5 4 层叠加 = 真正护城河

| 层级 | 护城河深度 | 复制度 | 你的独占性 |
|---|---|---|---|
| L1 数据 | 浅 | 100% 复制 | 0%（人人能爬） |
| L2 标签 | 中 | 60% 复制 | 30%（标准是你定的） |
| L3 解读 | 深 | 10% 复制 | 90%（个人风格） |
| L4 推演 | 极深 | 1% 复制 | 99%（经验 + 视野） |

**关键判断：你的产品必须做到 L3+ 才值 ¥0.5/次，L1+ L2 只值 ¥0.01/次。**

## 三、B2A 业务的"信任经济"特性

### 3.1 信任成本极高

一次错数据 = 毁掉所有调用：

- agent 主人：*"上次你说 X，我信了，结果错了"* → 永久弃用
- 信任恢复成本：10x 错数据成本

### 3.2 信任建立的 4 个锚点

| 锚点 | 怎么建立 |
|---|---|
| **准确率** | 100% 多源验证，错就承认 |
| **可追溯** | 每条数据带 source URL |
| **时效** | 24h 内必须出，否则不报 |
| **人设** | 实名背景（化名 + 透明背景） |

### 3.3 错误的代价

**L1 错误**：标错事件类别 → 用户感觉不专业
**L2 错误**：标错重要性 → 用户优先级错乱
**L3 错误**：解读错 → 用户决策错（**最致命**）
**L4 错误**：推演错 → 用户错失机会或踩坑（**最致命**）

**对策：**
- L3/L4 必须人工终审，AI 只能给候选
- 错就立刻发更正（透明 > 完美）
- 建立"判断置信度"字段，让 agent 主人自己加权

## 四、B2A 与 B2C 的本质差异

### 4.1 销售链路

**B2C（订阅制）：**
```
公众号/Newsletter → 吸引关注 → 转化订阅 → 续费
       ↓
需要教育用户"为什么要订阅"
```

**B2A（API）：**
```
agent 主人 → 找到 API 文档 → 测试调用 → 集成到 agent → 持续付费
       ↓
agent 自己集成，不需要"教育"
```

### 4.2 客户获取成本（CAC）

| 渠道 | B2C | B2A |
|---|---|---|
| 内容获客 | ¥10-50/人 | ¥0.1-1/调用 |
| 搜索获客 | ¥50-200/人 | 几乎为 0 |
| 口碑获客 | ¥20-100/人 | 几乎为 0 |

**B2A 的 CAC 接近 0**——一旦上架 MCP / Skill 市场，agent 自动发现。

### 4.3 客户生命周期价值（LTV）

| 维度 | B2C | B2A |
|---|---|---|
| 单客户 ARPU | ¥39-200/月 | ¥10-1000/月 |
| 留存周期 | 6-12 个月 | 12-36 个月 |
| 流失率 | 5-10%/月 | 1-3%/月 |

**B2A 的 LTV 是 B2C 的 5-10 倍**——agent 不会"忘了订阅"。

## 五、护城河的时间维度

### 5.1 短期（0-6 个月）

护城河来源：
- **先发优势**——你是这个品类的第一批
- **数据资产**——积累的事件库 + 标签库
- **人设 IP**——化名 + 个人风格

### 5.2 中期（6-18 个月）

护城河来源：
- **API 网络效应**——更多 agent 调用 → 更多数据 → 更好判断
- **客户数据**——哪些事件最常被问，反向优化采集
- **合作生态**——和 MCP 平台、agent 钱包深度合作

### 5.3 长期（18 个月+）

护城河来源：
- **行业标准制定权**——你定义的"重要性等级"成为事实标准
- **数据飞轮**——你的解读被引用 → 影响力 → 资源 → 更多解读
- **品牌资产**——AI Radar = AI 行业情报的代名词

## 六、核心推论

### 6.1 你必须做的 3 件事

1. **从 Day 1 就按"agent 可读"标准生产**
   - 哪怕初期没有 API 调用，也先把每条事件结构化
   - 这是未来 B2A 化的零成本切换

2. **建立"判断置信度"思维**
   - 每条 L3/L4 输出标 confidence（0-1）
   - 让 agent 主人自己决定权重
   - 这是 B2A 信任的关键

3. **从第一天就建数据资产**
   - 不是"每周发周报"
   - 是"每周发周报 + 同时入库事件库"
   - 3 个月后你的事件库就是别人抄不去的护城河

### 6.2 你不能做的 3 件事

1. **不能只做 L1+L2**——会被通用搜索和免费数据库卷死
2. **不能不做质量校验**——一次错数据毁掉所有调用
3. **不能等"准备好了再上"**——数据飞轮需要时间启动，越早上越好

## 七、一句话总结

**B2A 卖情报的本质 = 卖"对 agent 决策的边际价值"= 卖"真人判断的机器可读版本"。**
**护城河 = L3 解读 + L4 推演 + 数据资产 + 行业标准制定权。**
**关键动作 = 从 Day 1 就结构化 + 标置信度 + 建数据飞轮。**
