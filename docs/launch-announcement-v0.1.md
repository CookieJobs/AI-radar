## 长版（500-800 字）

### 为什么是现在

2026 年的下半年，一个你等了很久的东西要落地——**微信和支付宝的 agent 钱包**。

到时候你手机里的 agent 不只是"陪你聊"，它能直接帮你付钱、买东西、订服务、付 SaaS 月费。这意味着 agent 第一次成为一个**真正的消费实体**——它会有自己的"账本"，有自己的"预算"，会有"愿意为什么付费"的偏好。

而 agent 钱包里的钱花在哪，决定了未来 5 年 AI 应用的格局。

但 agent 自己有两个明显的弱点：

1. **它不刷 RSS**——你让它每天扫 50 个信源，token 烧光，效率归零
2. **它没判断力**——看到 OpenAI 发新模型，它只能复述标题；它说不清"这对你的业务意味着什么"

这是 Radar 做的事——**把"事件 + 解读 + 推演"打包成 agent 能直接调用的 API**。agent 调一次 `GET /v1/events`，拿走 200 字事实、200 字专家判断、100 字未来 30 天推演，喂给它的主人。

### 为什么是 API，不是周报

我试过做订阅周报。一周 8-12 小时，公众号/飞书/即刻/X 四平台分发，引流、转化、留存——每一个环节都在消耗一个 AI 产品经理的精力。

后来我想明白了：

- **周报是个人 IP 时代的产物**——作者出名、读者追更、平台分账
- **agent 时代不需要周报**——agent 7×24 在线，缺的是"我每次问都能拿到最新"

所以 Radar 砍掉了所有"给人读"的东西。**不写公众号、不出周刊、不做邮件订阅**。所有精力压在一件事上：让 agent 调一次得到的答案，比你自己刷两小时 RSS 强。

### 为什么是"我"做

我是 AI 产品经理出身，在教育大厂做 AI 学习产品，现在在尝试 OPC 模式。

我每天刷 50 个信源是基本功，但即使是我，**也会漏掉关键事件、错判信号、低估推演**。MCP 6 月 17 日发了 v1.0 官方规范——我读到的时候已经过了一天；OpenAI 融资估值细节，我看到新闻才知道。

我把自己读信、写解读、做推演的工作流**沉淀成了一个 API**。

**人写判断，AI 做摘要**——这是 Radar 不可替代的地方。大模型自己变实时了，事实层（L1）会越来越不值钱；但**判断层（L3）和推演层（L4）需要跨域视角 + 价值观 + 实战经验**——这些是 LLM 给不出的，至少现在给不出。

### 你可以怎么用

**如果你做 OPC**，让你的运营/调研 agent 接上 Radar——它能持续喂给你"这个赛道过去一周发生了什么、为什么重要、接下来会怎样"，你不用再花两小时自己刷。

**如果你是 AI 产品经理**，让你的研究 agent 接上 Radar——你准备 PRD、做竞品分析、写 OKR 的时候，直接让 agent 调最新事件来补上下文。

**如果你是早期投资人**，让你的尽调 agent 接上 Radar——你扫赛道、看竞对、判断窗口期的时候，让 agent 先帮你过一遍"这个方向过去 30 天的所有高 impact 事件"。

### 接入方式

- **REST**：`http://<your-host>/v1/events`（限速 60/min + 1000/day，冷启动期免费）
- **MCP**：Claude Desktop / Cursor / Cline 配 1 段 JSON 就能用
- **教程**：[docs/mcp-integration-guide.md](./mcp-integration-guide.md)

### 我的承诺

- **判断和推演 100% 我手写**——LLM 只做格式转换，不替我下结论
- **每周 5-8 条事件**——质量优先，不为日更而注水
- **冷启动期**——完全免费、限速合理、欢迎提反馈

### 别期望的事

- ❌ 这不是研报替代品——我不做行业全景图
- ❌ 这不是新闻聚合——你想要的"今天所有事"我给不了
- ❌ 这不是投资建议——推演不构成买卖依据
- ✅ 我给你的是"**高信号密度 + 跨域视角 + 可行动**"的判断

## 写作说明（给未来的我）

- **不暴露公司名**——只写"教育大厂"，"OPC 模式"暗示了个人身份
- **3 个 agent 主人场景都用动词开头**——"让你的 XX agent 接上"——降低想象成本
- **结尾"别期望的事"是关键**——主动划边界比"什么都做"更显专业
- **语气**：沉着、不营销、承认局限——这是 6 轮讨论里你强调的"专业但不死板"
- **字数**：约 750 字（含 3 句版），符合 brief 要求
